# wiki-sync 产品规划

## 一句话定位

把 Claude Code、Codex 等 AI Agent 的对话记录，一键导入 Obsidian LLM-WIKI 知识库。

## 解决的问题

和 AI 聊完就没了。对话记录躺在 `~/.claude/projects/` 的 JSONL 文件里，想找也找不到，
想回顾也不方便。如果有 LLM-WIKI 知识库，理论上能把每次对话变成可检索、可链接的 wiki 页面，
但手动搬运太麻烦。

**wiki-sync** 把这步自动化。

## 目标用户

在 Obsidian 中使用 LLM-WIKI 知识库，并且日常用 Claude Code / Codex 等 AI Agent 工作的人。

## 核心功能

### V0.1 — 最小可用

| 功能 | 描述 |
|------|------|
| 对话发现 | 自动扫描 Claude Code 的 sessions-index.json，列出可导入的对话 |
| 单条导入 | 把指定对话转为 LLM-WIKI 格式的 Markdown 文件，存入 vault |
| 去重保护 | 已导入的对话不会重复导入 |
| Index 更新 | 新页面自动加入 wiki/index.md 的索引 |
| Log 更新 | 导入操作记录到 wiki/log.md |

### V0.2 — 批量与自动化

| 功能 | 描述 |
|------|------|
| 批量导入 | `--all` 一键导入全部历史对话 |
| 自动模式 | 配置 Claude Code hook，对话结束自动导入 |
| 增量摘要 | 不只是全文转录，生成对话主题摘要和关键结论 |

### V0.3 — 多 Agent 支持

| 功能 | 描述 |
|------|------|
| Codex 适配 | 支持读取 Codex 的对话存储格式 |
| ChatGPT 适配 | 支持导入 ChatGPT 导出的对话 JSON |
| 通用适配层 | 定义对话抽象模型，方便接入新 Agent |

## 技术形态

### 核心：Python CLI（单文件、零依赖）

```bash
wiki-sync list                  # 列出可导入的对话
wiki-sync import                # 导入最近一条
wiki-sync import --session <id> # 导入指定对话
wiki-sync import --all          # 批量导入
```

选择 Python 的理由：macOS 自带，零外部依赖即可运行，文本处理顺手。

### 封装：Claude Code Skill

在 Claude Code 中安装 skill 后，对话中输入 `/sync-wiki` 即可导入，
无需退出 Claude Code、无需手动操作。

### 安装方式

```bash
# 用户只需一条命令
curl -sSL https://raw.githubusercontent.com/ruanjingwei/wiki-sync/main/install.sh | sh
```

卸载：

```bash
rm -f ~/.local/bin/wiki-sync
```

## 对话 → Wiki 的映射规则

Claude Code 对话存储结构：

```
~/.claude/projects/-Users-<name>/
├── sessions-index.json          ← 所有对话的索引
├── <uuid>.jsonl                 ← 每条对话的完整记录（用户消息 + AI 回复 + 工具调用）
```

导入后，在 LLM-WIKI vault 中生成：

```
vault/
├── raw/sources/claude-<slug>.md    ← 对话全文（原始素材，只读）
├── wiki/sources/claude-<slug>.md   ← 结构化页面（frontmatter + 摘要 + 要点）
├── wiki/index.md                   ← 追加索引条目
├── wiki/log.md                     ← 追加操作日志
└── .llm-wiki/conversations.json    ← 已导入记录（去重用）
```

### wiki 页面模板

```markdown
---
type: source
title: <对话标题>
tags: [claude-conversation, <自动提取的标签>]
related: []
created: YYYY-MM-DD
updated: YYYY-MM-DD
authors: [Claude Code]
---

# <对话标题>

## 对话信息
- **日期**: YYYY-MM-DD
- **消息数**: N
- **模型**: <model-name>

## 主题
<AI 自动生成的摘要>

## 关键结论
- ...
- ...

## 原文
见 [[raw/sources/claude-<slug>]]
```

## 和同类产品的区别

| | wiki-sync | Mem0 | 手动复制粘贴 |
|---|---|---|---|
| 目标 | 对话 → Obsidian wiki | 通用记忆层 | — |
| 存储位置 | 你的本地 vault | 云端 | 你的 vault |
| 格式 | LLM-WIKI schema | 自有格式 | 随意 |
| 自动化 | CLI + skill | API | 无 |
| 数据主权 | 完全本地 | 云端服务 | 完全本地 |

## 技术依赖

- **Python 3.9+**: macOS 自带，其他系统需安装
- **LLM-WIKI**: 用户的 Obsidian vault 必须已初始化 LLM-WIKI 结构（有 `.llm-wiki/` 目录和 `wiki/` 目录）
- **Claude Code** (可选): 仅使用 Skill 封装时需要
